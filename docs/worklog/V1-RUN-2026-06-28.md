## Goal
Turn NOLA building permits into a **labeled dataset of plan pages for commercial flooring takeoff**
(partner: Elite Installation). This run builds the **labeling loop** — Claude labels → human reviews/
corrects → becomes training data — and proves it at the 5 → 25 gate.

## The pipeline (locked-in stage order)
```
Collect → Prep Evidence → Claude Label → Human Review → Full-Extract process docs → Train
```
- **Two extractions:** cheap *prep* (per-page text, what Claude reads) vs *full* extraction
  (image + text JSON + vector JSON → S3/RDS).
- **Document label gates full extraction; page label does NOT.** Every `process` doc gets all
  pages extracted; page labels are training metadata, not a page-skip filter (yet).

## Key decisions & learnings
1. **Name-triage before download.** A cheap pass over filenames sends obvious MEP/plumbing/
   electrical/fire/receipts straight to `raw_only` *without downloading*; only architectural/
   ambiguous docs are pulled. On Planet Fitness this dropped **17 of 22 docs** before download
   (no more ~500 MB of MEP). Filenames are trusted only where they don't lie; ambiguous → read content.
2. **The concurrency trap.** This Codespace has **2 cores**, and the workflow engine caps agents at
   `cores − 2` = **1 at a time** → labeling ran serial (barely using credits). Fix: the cap is
   **per-workflow**, so fanning out across **7 parallel workflows** lifted it from 1 → ~18 concurrent
   agents. (Cleaner long-term fix: bump the Codespace to 8+ cores.)
3. **8-vCPU extraction box.** Resized the runner `t3.large (2 vCPU)` → `m7i.2xlarge (8 vCPU)` and ran
   extraction 8 workers wide (~4× faster). 8 vCPU is the account's On-Demand ceiling.
4. **Durable files = no loss.** Labels land in `data/labels/lb_<permit>.json` (durable) the instant a
   project finishes; a store loop pushes them to RDS via the EC2 box (reliable, in-region).
